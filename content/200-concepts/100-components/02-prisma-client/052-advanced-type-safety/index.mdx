<TopBlock>

Prisma Client 生成的代码包含几个有用的类型和实用程序，你可以使用这些类型和实用程序使你的应用程序更加安全。此页面描述了利用它们的模式。

</TopBlock>

## 导入生成类型

你可以导入 `Prisma` 命名空间，并使用点符号访问类型和实用程序。下列示例显示了如何导入 `Prisma` 命名空间并将其用于访问和使用  `Prisma.UserSelect` [generated type](#what-are-generated-types) 生成的类型：

```ts
import { Prisma } from '@prisma/client'

// Build 'select' object
const userEmail: Prisma.UserSelect = {
  email: true,
}

// Use select object
const createUser = await prisma.user.create({
  data: {
    email: 'bob@prisma.io',
  },
  select: userEmail,
})
```

另请参阅: [使用 `Prisma.UserCreateInput` 生成类型](../crud#create-a-single-record-using-generated-types)

## 生成的类型是什么? {#what-are-generated-types}

生成的类型是源自模型的类型脚本类型。你可以使用它们创建键入对象，你传递到顶级方法，如 `prisma.user.create(...)` 或  `prisma.user.update(...)`，或选项，如 `select` 或 `include`。

例如， `select` 接受 `UserSelect` 类型的对象。根据模型，其对象属性与 `select` 语句支持的对象属性相匹配。

下面的第一个选项卡显示了 `UserSelect` 生成的类型以及对象上的每个属性如何具有类型注释。第二个选项卡显示生成的 schema 模型。

```ts
type Prisma.UserSelect = {
    id?: boolean | undefined;
    email?: boolean | undefined;
    name?: boolean | undefined;
    posts?: boolean | Prisma.PostFindManyArgs | undefined;
    profile?: boolean | Prisma.ProfileArgs | undefined;
}
```

</tab>

model User {
  id      Int      @id @default(autoincrement())
  email   String   @unique

在类型脚本中，当你声明一个变量并添加类型注释来描述变量的类型时 [类型注释](https://www.typescriptlang.org/docs/handbook/2/everyday-types.html#type-annotations-on-variables) 的概念。请参阅下示例。

```ts
const myAge: number = 37
const myName: string = 'Rich'
```

这两个变量声明都有一个类型注释，分别指定它们是什么原始类型，`number` 和 `string`。大多数情况下不需要这种注释，因为 TypeScript 将根据变量的初始化方式推断变量的类型。在上面的示例中， `myAge`  是用一个数字初始化的，因此 TypeScript 猜测它应该是一个数字。

回到 `UserSelect` 类型，如果要在创建的对象 `userEmail` 上使用 `.` 表示法，则可以访问 `User` 模型上的所有字段，这些字段可以使用 `select` 语句进行交互。

```prisma
model User {
  id      Int      @id @default(autoincrement())
  email   String   @unique
  name    String?
  posts   Post[]
  profile Profile?
}
```

```ts
import { Prisma } from '@prisma/client'

const userEmail: Prisma.UserSelect = {
  email: true,
}

// properties available on the typed object
userEmail.id
userEmail.email
userEmail.name
userEmail.posts
userEmail.profile
```

在同一模块中，这样在你使用 `include` 语句之后 的对象就可以访问到那些属性  。

```ts
import { Prisma } from '@prisma/client'

const userPosts: Prisma.UserInclude = {
  posts: true,
}

// 类型化对象上可用的属性
userPosts.posts
userPosts.profile
```

> 有关可用不同类型的更多信息，请参阅 [模型查询选项](../../../../reference/api-reference/prisma-client-reference#model-query-options) 。
